## 2. 文档目标

本文用于冻结“建筑学长 AI创作无限画布 Agent”的真实产品形态，作为后续本项目复刻与功能验收的唯一 PRD 基线。重点不是抽象出一个“类似功能”，而是把真实页面中已经观察到的布局、按钮、二级菜单、右键菜单、弹窗、联动规则、状态切换和边界条件完整写清楚。

本文特别强调以下 5 个产品原则：

1. 画布不是简单图片容器，而是“节点化创作工作台”。
2. 生成不是覆盖式改图，而是“派生式生成”。
3. 选择、锁定、发送至对话是三层不同语义，不能混为一谈。
4. 画布、底部控制台、右侧创作记录之间存在强状态联动。
5. 节点血缘、历史记录和资产回流共同构成可追溯创作链。

## 4. 产品定位与核心定义

### 4.1 产品定位

建筑学长的 AI创作无限画布 Agent，是一个围绕“建筑效果图 / 建筑演示图 / 方案迭代 / 多轮分叉创作”组织起来的无限画布工作台。它不是传统聊天窗口，也不是传统图片管理器，而是把以下对象统一放在一张无限画布中：

- 原始底图
- 局部编辑副本
- 多图拼接结果
- 生成中的占位节点
- AI 生成结果图
- 节点之间的父子血缘连线
- 当前会话输入上下文
- 历史创作记录

### 4.2 核心产品语义

1. 节点语义：
   - 每一张图片都应被视为独立节点，而不是仅仅一段聊天附件。
2. 派生语义：
   - 新结果默认从旧节点派生，形成树状分支。
3. 输入语义：
   - 画布选中用于声明“当前候选输入”。
   - 点击输入框用于声明“锁定当前输入”。
   - `发送至对话` 用于把当前节点显式提升为对话附件上下文。
4. 追溯语义：
   - 生成关系必须可视化，可从结果追溯到父节点与操作语义。

### 9.1.1 `形状` 的联动规则

1. 点击左侧 `形状` 后，顶部立即出现独立的形状样式工具条；它与图片选中后的悬浮条不是同一套 UI。
2. `shape flyout` 本体不是带说明文字的卡片列表，而是 `3 + 2` 的 icon-only 按钮网格。
3. 当仍处于“已激活形状工具、尚未落图”的阶段时，顶部工具条处于 `tool mode`，至少暴露 `描边 / 填充 / 线宽`。
4. 当单选已有的矩形、圆形、菱形、箭头或直线时，顶部工具条切到 `selection mode`，除 `描边 / 填充 / 线宽` 外，还出现 `W / H` 宽高编辑。
5. 选中已有形状时，画布上方出现的是“形状编辑工具条”，不复用图片选中后的 `编辑 / 涂鸦 / 文字 / 查看大图 / 下载` 悬浮条。
6. 右键菜单与该工具条互斥；右键只打开上下文菜单，不额外唤起图片/形状悬浮条。

### 14.2 `发送至对话` 的强语义

该动作与普通选中不同，PRD 必须单独定义：

1. 普通选中：
   - 只是让控制台显示当前图片缩略图。
2. `发送至对话`：
   - 会把当前节点显式送入对话链路，并在当前选中仍存在时保留该图片 chip。
   - 若随后点击画布空白处清除选中，2026-04-16 follow-up 审计显示该 chip 会消失。
   - 因而现阶段不能把它定义为“独立持久附件状态”；更准确的说法是“显式发送动作加强了当前选区与对话的绑定，但该绑定仍受当前选中态影响”。
3. 展开右侧面板时：
   - 若当前选中未被清掉，记录区会围绕该图片/该轮操作展示相关卡片。

## 17. 底部控制台

### 17.1 控制台定位

底部控制台是全页面最高频的 AI 交互中枢。它不是单纯输入框，而是承载以下职责：

1. 当前选中图片的可视化上下文。
2. Prompt 输入。
3. 模型与参数选择。
4. 模板调用。
5. 发送与生成。

### 17.3 控制台状态规则

1. 折叠态：
   - 控制台位于底部居中。
2. 右侧面板展开态：
   - 控制台不消失，而是变为右下 docked composer。
3. 单图普通选中态：
   - 控制台先显示 1 个待选缩略图。
   - 待选态随当前选区变化，不自动持久化。
4. 单图确认态：
   - 只有输入框 focus 或显式 `发送至对话` 后，图片才升级为正式附件。
5. 多图选中态：
   - 自动显示多个待选缩略图，并允许调整输入顺序。
6. 显式 `发送至对话`：
   - 在不取消当前选中的前提下，面板切换不应清掉附件 chip。
   - 沉浸式右侧会短暂出现围绕该引用图的快捷记录卡；当用户回到输入框并完成正式附件确认后，该快捷卡应让位给正式附件显示。
7. 附件删除：
   - 待选态可随选区丢失而自动消失。
   - 已确认附件只能由用户手动点 `x` 删除。

## 19. 局部编辑与预处理流程

### 19.1 局部编辑入口

A 级证据可确认单图悬浮条包含 `编辑 / 涂鸦 / 文字`，B 级教学文档补充其核心业务意义如下：

1. `编辑`：
   - 进入专注式局部修改工作态。
2. `涂鸦`：
   - 用于直接在图上进行自由笔触标注或涂抹。
3. `文字`：
   - 用于在图上叠加文字标注。

### 19.2 遮罩编辑规则

教学文档明确给出如下规则，应纳入 PRD：

1. 进入编辑态后，出现笔刷控制条。
2. 用户通过笔刷在目标区域涂抹，遮罩为半透明红色。
3. 保存存在两条语义不同的分支：
   - `保存`：覆盖原节点。
   - `保存为副本`：原图保留，新建带遮罩的新节点。

### 19.3 多图拼接与合并

教学文档把该能力定义为核心业务之一：

1. 可以先在画布中导入多张素材。
2. 自由拖拽、缩放、排版。
3. 框选目标素材后执行 `合并图层`。
4. 合并结果变为新的单一输入节点，继续参与生成。

### 19.4 风险点

1. `合并图层` 若不可逆，需要明确提示。
2. 若支持撤销，撤销范围必须覆盖合并前状态。

## 20. 生成流程与节点血缘

### 20.1 生成触发

1. 控制台输入 Prompt。
2. 配置模型、比例/参数、分辨率。
3. 点击发送按钮触发生成。

### 20.2 已确认画布表现

根据 `content-1-baseline.png`、`opened-card-from-home.png` 和教学文档：

1. 画布中会保留父节点。
2. 新结果默认生成在父节点下方。
3. 父节点与子节点之间用带箭头的曲线相连。

### 20.3 教学文档补充的加载态

1. 生成中先出现骨架屏或 Loading 占位框。
2. 结果完成后用真实图片替换占位框。
3. 这意味着“生成中”本身也是画布中的可见状态，而不是只在按钮上转圈。

### 20.4 连续迭代规则

1. 新生成结果可以再次被选中。
2. 新结果可以再次局部编辑、保存为副本、重新输入 Prompt、再次生成。
3. 多轮生成后形成树状创作链。

## 24. 画布、控制台、右侧面板的联动映射

这是本文最重要的联动规则，后续实现必须严格遵守。

| 用户动作 | 画布变化 | 控制台变化 | 右侧面板变化 |
| --- | --- | --- | --- |
| 单击 1 张图片 | 节点出现单选边框与悬浮工具条 | 出现 1 个缩略图附件 | 默认不自动展开 |
| 框选多张图片 | 出现多选边框与聚合工具条 | 出现多个缩略图附件 | 默认不自动展开 |
| 点击输入框 | 当前选择进入锁定输入态 | 输入焦点激活 | 不变 |
| 右键 `发送至对话` | 选区被显式送入对话链路 | 若选区仍保留，chip 继续存在；取消选中后 chip 消失 | 后续展开时可围绕该图显示记录卡 |
| 点击 `对话` | 不改变画布选区 | 控制台从居中改为右下 docked | 右侧创作记录展开 |
| 点击 `收起` | 不改变画布内容 | 控制台回到底部居中 | 右侧面板关闭 |
| 点击空白处取消选中 | 清除当前选框 | 对应图片 chip 消失 | 不强制关闭面板 |
| 点击 `生成` | 父节点下方出现 Loading，占位后替换成子节点并连线 | 维持当前输入上下文 | 增加新记录卡、文件列表状态 |

## 25. 异常与边界情况

### 25.1 必须处理的边界

1. 大画布性能：
   - 多张大图同时存在时，需要懒加载与降级渲染。
2. 连线实时跟随：
   - 节点拖拽时，箭头必须实时更新。
3. 合并图层风险：
   - 如不支持撤销，必须做风险提示。
4. 输入焦点冲突：
   - 已输入文字后切换底图，不应丢失已有文字。
5. 遮罩对齐：
   - 红色遮罩显示层与后端提交 mask 必须精确一致。
6. 权限受限：
   - 2K / 4K / 高清增强等高阶能力应给出明确反馈。
7. 历史定位失败：
   - 若从右侧记录点击某图无法定位画布节点，需给出失败提示。
8. 悬浮条触发边界：
   - 图片悬浮条只允许左键选中触发；右键必须只开上下文菜单。
9. 待选/确认状态边界：
   - 输入框激活前的参考图不能伪装成正式附件；输入框激活后又不能被取消选中意外清空。
10. 形状编辑边界：
   - 形状激活态与形状选中态必须切换同一套顶部工具条，且不能误用图片悬浮条。

### 25.2 当前已知待复核项

以下功能在截图中有存在迹象，但未通过本轮浏览器直连完成语义确认：

1. 模板库的完整三级条目。
2. `查看大图` 的基础按钮集合已经确认；当前待补的是是否还存在更深的高清增强 / 无损放大分支与对应权限状态。
3. `导出` 的菜单语义已在 2026-04-17 follow-up 审计中收敛：
   - 右键菜单 DOM 中 `导出` 项未出现 `aria-haspopup`、`aria-expanded` 或任何子菜单节点。
   - 点击 `导出` 时菜单会直接关闭，未出现 PNG / JPG 级联面板。
   - 结合本项目本地实现，当前冻结结论为“直接导出动作”，而不是“格式二级菜单”。
   - 当前剩余不确定点只是在自动化环境下是否能稳定抓到真实浏览器下载事件，不再是产品信息架构待定。
4. 多图输入顺序箭头已在 2026-04-17 live follow-up 审计中收敛，不再视为开放项：
   - 顺序箭头位于每个待选缩略图底边内部，而不是缩略图外部；
   - 首尾箭头保留禁用态，不会直接隐藏；
   - 右上关闭按钮只移除输入上下文中的该图，不影响画布当前选区；
   - 当前未看到额外文字 hover 提示，也不应在本地凭空补做说明气泡。

## 26. 结论

建筑学长的 AI创作无限画布 Agent，本质上是“基于节点血缘的建筑图像创作工作台”。其产品复杂度不在于单个按钮数量，而在于三条主线必须同时成立：

1. 画布主线：
   - 上传、布局、编辑、合并、生成、分叉、追溯。
2. 对话主线：
   - 选择、锁定、发送至对话、模板填充、模型参数配置、生成。
3. 记录主线：
   - 右侧创作记录、结果预览、快捷改写、文件列表、剩余次数与充值。

后续复刻时，不能把这些能力拆成“一个聊天页 + 一个白板页 + 一个图库页”的松散组合，而必须保持三者在同一张无限画布上的同步联动。
